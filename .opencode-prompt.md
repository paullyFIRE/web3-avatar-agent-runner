Issue title:
[HIGH] Session cookie domain uses includes() substring matching

Issue body:
## Summary

Domain matching in session cookie logic uses `allowedDomains.find((d) => requestDomain.includes(d || ""))`. The includes() substring match means alloy.tools matches attacker-alloy.tools, and cordex.finance matches cordex.finance.attacker.com or evilcordex.finance.

## Affected file

packages/server/src/lib/session.ts - lines 281-288

## Current code

```typescript
const domain = requestDomain && allowedDomains.find((d) => requestDomain.includes(d || ""));
```

## Impact

- An attacker who controls evil-alloy.tools could potentially receive session cookies
- Cookie injection via domain confusion
- While the Domain attribute limits scope, the matching is too permissive

## Required actions

Replace includes() with exact hostname matching:

```typescript
const parsed = new URL(
  requestDomain.startsWith("http") ? requestDomain : `https://${requestDomain}`
);
const matched = allowedDomains.find(
  (d) => parsed.hostname === d || parsed.hostname.endsWith(`.${d}`)
);
```

## Test case

```
it("rejects evil subdomain match", () => {
  const allowed = ["alloy.tools", "cordex.finance"];
  expect(allowed.find((d) => "evil-alloy.tools".includes(d))).toBe("alloy.tools");
  // With fix, should not match
});
```
